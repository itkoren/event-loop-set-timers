About The setTimeout Frequency Demo

This page tests the frequency of setTimeout in the browser. Javascript applications use setTimeout as a mechanism to 'yield' to the browser so that the browser can repaint. Some browsers implement a 15ms setTimeout minimum (based on the HTML4 spec) but the newer browsers implement a 4ms setTimeout minimum (based on the HTML5 spec). This page measures the setTimeout lag and discover the browser's minimum interval.

The original demo was created by Mike Belshe from Google as part of a demo he made for browser timers. I (@itkoren) adapted the demo to demonstrate the event loop and the importance of using timers in order not to starve it - the source code can also be found on GitHub.
